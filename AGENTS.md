## Recent Learnings

- **Tool-use scoring needs explicit first-discovery vs state-reuse guidance -> Add per-turn `tool_use_guidance` and surface it in the judge prompt so lookup-required catalog introductions stay strict while later turns can reuse already-established item facts without redundant lookups -> Prevents grocery-style evals from penalizing efficient session-state reuse and makes the intended contract legible to the judge.**
- **Realtime tool continuation needs two server-side gates -> Wait for both `conversation.item.added`/`conversation.item.done` on the injected `function_call_output` item and the prior `response.done` before sending deferred `response.create` -> Prevents missing a fast tool-output ack or fast response completion while the websocket send is still in flight, which can otherwise leave the continuation racing stale conversation state.**
- **Appointment-bench LLMAJ KB false fails are not stable under repeated replay -> When `gpt-5.2` marks appointment-bench turns 9 or 23 as `kb_grounding=false`, verify with repeated same-prompt replays before changing GT or prompt wording -> In a March 11, 2026 experiment on run `runs/appointment_bench/20260311T100636_gpt-realtime-1.5_448966ba`, replaying the current rehydrated LLMAJ prompt for turns 9 and 23 10 times at `reasoning_effort=none` and 10 times at `low` produced `kb_grounding=true` on 20/20 runs for both turns; `low` occasionally added a separate turn-23 completeness/state penalty, but never reproduced the original grounding miss.**
- **Rehydrated runtime summaries can overstate completion after a failed final turn -> Verify `transcript.jsonl` and judged-row coverage even when `runtime.json` says `failed_turns=[]` and all turns succeeded -> In the March 11, 2026 run `runs/event_bench/20260311T100636_gpt-realtime-1.5_0ba2711a`, `runtime.json` reported 29/29 success, but `transcript.jsonl` and `openai_judged.jsonl` stopped at turn 27 after turn 28 hit `no close frame received`, `Error connecting: [Errno 8] nodename nor servname provided, or not known`, and `session was not ready before rehydration seeding`.**
- **Rehydrated reconnect fallback must preserve tool results -> Mirror `function_call_response` payloads into reconnect history and render them back into the fallback prompt -> Prevents fresh-session retries from losing benchmark state after websocket reconnects.**
- **Grocery-bench rehydrate can complete cleanly while failures stay mostly model-side -> Treat `grocery_bench --rehydrate --disable-vad` runs with full `30/30` transcript coverage, `30/30` judged rows, and `turn_taking_failures=[]` as scoreable, then bucket remaining `gpt-realtime-1.5` misses into missed lookup/tool-completeness turns, pre-order state confusion, and post-order edit mistakes instead of runtime artifacts -> In the March 11, 2026 run `runs/grocery_bench/20260311T100643_gpt-realtime-1.5_364e7750`, the artifacts were clean, but turns 2/5/17/20/23 missed required lookup completeness, turn 19 wrongly claimed it could not edit the in-progress order yet, and turns 26-27 mishandled the post-order modification logic.**
- **Manual no-VAD retries need explicit state reset -> Clear OpenAI Realtime manual commit/response flags before replaying a turn's audio on retry -> Prevents retried `--disable-vad` turns from being dropped and mis-scored as model failures because duplicate stop events are ignored.**
- **Appointment-bench rehydrate skip-turn-taking needs manual bucket correction -> On `appointment_bench` rehydrated `gpt-realtime-1.5` runs, treat raw `kb_grounding` fails on turns 9 and 23 as likely judge grounding mistakes and subtract the expected closing `end_session` turn before interpreting `MODEL_ENDED_SESSION` counts -> In the March 11, 2026 run `runs/appointment_bench/20260311T100636_gpt-realtime-1.5_448966ba`, the artifacts were clean `25/25`, all required tool turns were on time, the only clear real model errors were turn 1's missed Thursday schedule constraint and turn 17's premature session close, while turns 2/9/23 were better explained by strictness or judge-side grounding issues.**
